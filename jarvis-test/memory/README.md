# memory/ — дневники Агента

В этой папке Агент ведёт **дневники работы по дням**. По одному файлу на день в формате `YYYY-MM-DD.md`.

Например: `2026-04-29.md`, `2026-05-01.md`.

## Что туда писать

Не «всё подряд». Только три вещи:

1. **Что сделано** — какие задачи закрыли, какие файлы создали или поправили
2. **Какие решения приняли** — особенно если решение спорное или важное
3. **Что осталось на потом** — TODO, открытые вопросы, идеи на завтра

Дневник — рабочий журнал, не отчёт. Пишите коротко, телеграфно.

## Формат файла

```markdown
# 2026-04-29

## Сессия 1 (10:30)
Тема: настройка нового лендинга

### Сделано
- Скачали шаблон, выбрали стиль
- Настроили текст на главной

### Решения
- Cтраницы делаем без анимаций, ради скорости
- Подключаем Тильду, не делаем с нуля

### TODO на потом
- Подобрать иконки для раздела «Услуги»
- Настроить домен
```

Если за день несколько сессий — добавляйте `## Сессия 2 (15:00)` ниже первой.

## Зачем это нужно

Через неделю Вы откроете проект — и Агент **прочитает последние дневники**, поймёт где Вы остановились, что было решено, что Вы обещали себе доделать. Без дневников каждая новая сессия начинается с нуля — Агент Вас не помнит.

## Когда Агент создаёт файл

- В **начале новой сессии** — если файла за сегодня ещё нет, создаёт пустой по шаблону.
- **По ходу работы** — дописывает в раздел «Сделано» после каждого завершённого шага.
- **В конце сессии** — подводит итог в «Решения» и «TODO».

## Что НЕ писать в дневник

- Долгосрочные факты о Вас → они идут в `MEMORY.md` (раздел «Кто я» / «Проекты»)
- Большие справочники и инструкции → они идут в `knowledge/`
- Детальный код или контент задач → они в самих файлах проекта, в дневнике только ссылка

## Как просматривать дневники

Это просто markdown-файлы — открывайте в VS Code, GitHub, любом текстовом редакторе. Пролистайте за месяц — и видите всю историю работы над проектом.
