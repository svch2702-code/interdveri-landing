# knowledge/ — база знаний Агента

В эту папку складываются **справочники, инструкции, чек-листы, регламенты** — всё что Агент должен читать **по запросу**, но не каждую сессию.

## Чем `knowledge/` отличается от `MEMORY.md`

| `MEMORY.md` | `knowledge/` |
|---|---|
| Короткие факты | Длинные документы |
| «Я работаю на Beget VPS, IP такой-то» | Инструкция «Как развернуть бота на Beget» |
| Читается **в начале каждой сессии** | Читается **когда тема всплыла** |
| Один файл, ≤ 200 строк | Много файлов, любого размера |

Правило: если что-то занимает 1-2 строки — это в `MEMORY.md`. Если это документ на 50+ строк — это `knowledge/`.

## Что обычно складывают сюда

- **Брендбук** — стиль текстов, дизайна, голос автора (`brandbook.md`)
- **Деплой-гайды** — как разворачивать проекты (`deploy-guide.md`, `vps-setup.md`)
- **Чек-листы** — запуск проекта, ревью кода, подготовка к эфиру (`launch-checklist.md`)
- **Шаблоны** — пост в блог, презентация, коммерческое предложение (`post-template.md`)
- **Регламенты команды** — как работаем с подрядчиками (`team-rules.md`)
- **Гайды по сервисам** — как пользоваться вашим стеком (`tilda-guide.md`, `getcourse-guide.md`)

## Формат файлов

Обычные `.md` файлы. Внутри — структурированно, но без избыточности. Принцип: «один файл — одна тема».

## Как Агент знает что читать

Один из двух способов:

1. **Через `MEMORY.md`** — в разделе «Ссылки на knowledge/» Вы (или Агент) дописываете когда какой файл читать. Например: *«Брендбук — `knowledge/brandbook.md`. Читать когда пишешь пост, оформляешь презентацию или делаешь любой контент от моего имени.»*

2. **Через прямой запрос** — Вы говорите Агенту: «Я добавил `knowledge/launch-checklist.md` — учти его при подготовке запуска». Агент прочитает и запомнит.

## Когда создавать новый файл в `knowledge/`

Хорошее правило: **когда тот же вопрос всплыл 2+ раза**. Если Вы дважды объясняли Агенту одно и то же — это сигнал что пора вынести в knowledge/.

## Примеры названий

```
knowledge/
├── brandbook.md              ← голос автора, стиль
├── deploy-guide.md           ← как разворачивать проекты
├── launch-checklist.md       ← запуск нового лендинга
├── post-template.md          ← шаблон поста в Telegram
├── meeting-rules.md          ← как мы проводим встречи с командой
└── ai-prompts-library.md     ← собранные удачные промпты
```

Названия — короткие, латиница, через дефис. Файл должен быть понятен по имени.
