# Чек-лист проверки практического воркшопа

Используй этот чек-лист для проверки материалов к практическим воркшопам внутри продукта (курс AI-Эксперт, фокус-группа, НейроКлуб).

**Отличие от продающих вебинаров:** здесь цель = НАУЧИТЬ, а не продать. Фокус на практике, результате и ощущении "Я уже умею".

---

## 📋 Как пользоваться

1. Проходи по каждому разделу последовательно
2. Ставь ✅ напротив выполненных пунктов
3. Если пункт не выполнен — исправляй перед финализацией
4. **Критические пункты** (❗) — обязательны

---

## 1. СТРУКТУРА СЕРИИ ВОРКШОПОВ

### 1.1 Общая структура (для серии из нескольких эфиров)

- [ ] ❗ Каждый воркшоп имеет чёткое название
- [ ] ❗ У каждого воркшопа определена основная цель
- [ ] ❗ Прописан результат: "На выходе у ученика есть..."
- [ ] Указаны ключевые инструменты и нейросети
- [ ] ❗ Есть логика перехода между воркшопами (каждый готовит к следующему)
- [ ] Определена эмоциональная цель каждого дня

### 1.2 Целостность серии

- [ ] Первый воркшоп создаёт "входной ВАУ-эффект"
- [ ] Каждый следующий воркшоп развивает навыки предыдущего
- [ ] Финальный воркшоп закрепляет всё и даёт мощный результат
- [ ] Нет "провисающих" воркшопов без практической ценности

---

## 2. СТРУКТУРА ОДНОГО ВОРКШОПА

### 2.1 Обязательные элементы

- [ ] ❗ **Название воркшопа** — понятное, мотивирующее
- [ ] ❗ **Цель занятия** — что ученик научится делать
- [ ] ❗ **Краткое содержание** — пошаговое описание этапов
- [ ] ❗ **Практика** — промпты по формуле RAFT на каждом шаге
- [ ] ❗ **Инструменты** — какие нейросети использовать
- [ ] ❗ **Результат на выходе** — конкретный артефакт (файл, навык, структура)
- [ ] **Роль куратора** — как поддерживает, какие комментарии даёт
- [ ] **Тон подачи** — профессиональный, воодушевляющий, простой

### 2.2 Хронометраж

- [ ] Определена общая длительность воркшопа
- [ ] Каждый этап имеет примерное время
- [ ] Практика занимает минимум 50% времени
- [ ] Есть время на вопросы и разборы

---

## 3. ВОВЛЕЧЕНИЕ И ВАУ-ЭФФЕКТ

### 3.1 Первые 5 минут

- [ ] ❗ Зацепка — почему это важно прямо сейчас
- [ ] ❗ Обещание результата — что будет в конце воркшопа
- [ ] Связь с предыдущим воркшопом (если серия)
- [ ] Энергия и настрой — участник понимает "это будет круто"

### 3.2 ВАУ-эффект

- [ ] ❗ ВАУ-момент в первые 10 минут (демонстрация возможности)
- [ ] ВАУ-момент каждые 15-20 минут
- [ ] Типы ВАУ:
  - [ ] "Вау, так просто?!"
  - [ ] "Вау, я уже сделал это!"
  - [ ] "Вау, это работает!"
  - [ ] "Вау, у меня получилось лучше чем ожидал!"

### 3.3 Мотивация

- [ ] ❗ Участник понимает ЗАЧЕМ ему это (связь с его жизнью/бизнесом)
- [ ] Есть истории успеха других учеников
- [ ] Ощущение "Я уже умею" после каждого блока
- [ ] Празднование маленьких побед ("Отлично! У вас получилось!")

---

## 4. ПРАКТИЧЕСКАЯ ЧАСТЬ

### 4.1 Промпты по формуле RAFT

Для каждого практического задания проверь наличие:

- [ ] ❗ **R — Role (Роль):** кем является ИИ в этом промпте
- [ ] ❗ **A — Audience (Аудитория):** для кого создаётся результат
- [ ] ❗ **F — Format (Формат):** в каком виде нужен результат
- [ ] ❗ **T — Task (Задача):** что конкретно нужно сделать

### 4.2 Качество промптов

- [ ] ❗ Промпты готовы к копированию (не нужно ничего додумывать)
- [ ] Есть плейсхолдеры для персонализации: `{ваша тема}`, `{ваш продукт}`
- [ ] Промпты протестированы и дают хороший результат
- [ ] Есть примеры готовых результатов

### 4.3 Пошаговость

- [ ] ❗ Каждый шаг описан максимально просто
- [ ] Шаги идут в логичном порядке
- [ ] Нет "скачков" — каждый шаг опирается на предыдущий
- [ ] Участник может повторить самостоятельно после воркшопа

---

## 5. ИНСТРУМЕНТЫ И НЕЙРОСЕТИ

### 5.1 Выбор инструментов

- [ ] ❗ Указаны конкретные нейросети для каждой задачи
- [ ] Выбраны доступные инструменты (бесплатные или с бесплатным лимитом)
- [ ] Есть альтернативы (если один инструмент не работает)
- [ ] Указаны ссылки на инструменты

### 5.2 Инструкции по работе

- [ ] Как зарегистрироваться/войти (если нужно)
- [ ] Где найти нужную функцию
- [ ] Куда вставить промпт
- [ ] Как сохранить/экспортировать результат

---

## 6. РЕЗУЛЬТАТ НА ВЫХОДЕ

### 6.1 Конкретный артефакт

- [ ] ❗ Чётко описано, что получит ученик (файл, документ, навык)
- [ ] Результат можно "пощупать" (не абстрактный)
- [ ] Результат сразу применим в жизни/бизнесе
- [ ] Участник понимает, как использовать результат дальше

### 6.2 Типы результатов

- [ ] Готовый документ (контент-план, скрипт, план)
- [ ] Настроенный инструмент (бот, автоматизация)
- [ ] Освоенный навык (умение делать X)
- [ ] Структура/шаблон для повторного использования

---

## 7. ПРОСТОТА ЯЗЫКА

### 7.1 Правило 10-летнего ребёнка

- [ ] ❗ Текст понятен без технического бэкграунда
- [ ] Нет жаргона без объяснения
- [ ] Сложные термины объясняются через метафоры
- [ ] Короткие предложения (не больше 15-20 слов)

### 7.2 Метафоры для нейросетей

- [ ] Нейросеть → "умный помощник"
- [ ] Промпт → "инструкция/рецепт для робота"
- [ ] Токены → "слова, которые понимает робот"
- [ ] Контекст → "память помощника"
- [ ] Генерация → "робот создаёт/придумывает"

### 7.3 Тон подачи

- [ ] Профессиональный, но не занудный
- [ ] Воодушевляющий, но не наигранный
- [ ] Поддерживающий ("У вас получится!")
- [ ] Наставнический ("Давайте попробуем вместе")

---

## 8. РОЛЬ КУРАТОРА/ПРЕПОДАВАТЕЛЯ

### 8.1 Поддержка участников

- [ ] Прописаны комментарии для типичных ситуаций:
  - [ ] "Если не получается — попробуйте..."
  - [ ] "Частая ошибка — ..."
  - [ ] "Если результат не тот — проверьте..."
- [ ] Есть подбадривающие фразы
- [ ] Есть ответы на частые вопросы

### 8.2 Активация интереса

- [ ] Куратор задаёт вопросы ("Кто уже попробовал?")
- [ ] Куратор просит показать результаты
- [ ] Куратор хвалит хорошие работы
- [ ] Куратор помогает с трудностями в реальном времени

---

## 9. ИНТЕРАКТИВ

### 9.1 Вовлечение аудитории

- [ ] ❗ Интерактив каждые 10-15 минут
- [ ] Типы интерактива:
  - [ ] "Поделитесь результатом в чат"
  - [ ] "Кто уже сделал — поставьте +"
  - [ ] "Какие вопросы возникли?"
  - [ ] "Покажите свой вариант"

### 9.2 Практика в реальном времени

- [ ] Участники делают вместе с преподавателем
- [ ] Есть пауза для самостоятельной работы
- [ ] Преподаватель показывает экран/демонстрацию
- [ ] Есть разбор работ участников

---

## 10. ТЕХНИЧЕСКИЙ ФОРМАТ (если презентация)

### 10.1 Формат файла

- [ ] Слайды в формате `===SLIDE===`
- [ ] Каждый слайд содержит:
  - [ ] ТИП: (контент/фото/чат/промпт)
  - [ ] ЗАГОЛОВОК:
  - [ ] ТЕКСТ:
  - [ ] СПИКЕР:
  - [ ] ВИЗУАЛ:

### 10.2 Особенности воркшоп-слайдов

- [ ] ❗ Промпты на отдельных слайдах типа `ТИП: промпт`
- [ ] Промпты выделены (контрастный фон, моноширинный шрифт)
- [ ] Пошаговые инструкции визуализированы
- [ ] Есть слайды с результатами (что должно получиться)
- [ ] Есть слайды с QR-кодами на инструменты

### 10.3 Слайды с промптами (важно для воркшопов!)

- [ ] ❗ Каждый промпт — на отдельном слайде `ТИП: промпт`
- [ ] Промпт готов к копированию (copy-paste ready)
- [ ] Плейсхолдеры выделены: `{ваша тема}`, `{ваш продукт}`
- [ ] Указано, куда вставлять промпт (ChatGPT, Claude, и т.д.)
- [ ] Описан ожидаемый результат

## 🔧 Что делать, если не проходит проверку

### Нет ВАУ-эффекта
→ Добавь демонстрацию в начале: "Смотрите, сейчас за 30 секунд создам..."

### Слишком сложно
→ Разбей на более мелкие шаги, добавь метафоры

### Промпты не работают
→ Протестируй заново, упрости, добавь контекст в промпт

### Участники не вовлечены
→ Добавь больше интерактива, попроси показать результаты

### Нет практики
→ Убери теорию, добавь "сделаем вместе"

### Результат абстрактный
→ Опиши конкретный артефакт: "файл с 10 идеями", "готовый скрипт продаж"
