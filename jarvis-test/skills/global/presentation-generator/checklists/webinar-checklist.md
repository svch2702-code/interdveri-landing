# Чек-лист проверки продающего вебинара

Используй этот чек-лист для проверки любой презентации и материалов к продающему вебинару.

---

## 📋 Как пользоваться

1. Проходи по каждому разделу последовательно
2. Ставь ✅ напротив выполненных пунктов
3. Если пункт не выполнен — исправляй перед финализацией
4. **Критические пункты** (❗) — обязательны, без них вебинар не работает

---

## 1. СТРУКТУРА ВЕБИНАРА (~400 слайдов)

### 1.1 Все блоки на месте

- [ ] ❗ **Блок 1: Вступление** (~20-30 слайдов)
  - [ ] Хайповый титульный слайд
  - [ ] Представление спикера с цифрами
  - [ ] Социальные доказательства (регалии, награды)
  - [ ] Продающий план вебинара с интригой

- [ ] ❗ **Блок 2: Кейсы** (~30-50 слайдов)
  - [ ] Результаты учеников/клиентов
  - [ ] Конкретные цифры (до/после)
  - [ ] Скриншоты, визуальные подтверждения
  - [ ] Разные ниши и ситуации

- [ ] ❗ **Блок 3: Актуализация — боли** (~40-60 слайдов)
  - [ ] Обоснование "почему именно сейчас"
  - [ ] Статистика масштаба проблемы
  - [ ] Боли каждого сегмента ЦА
  - [ ] Цена бездействия (что будет, если не решить)

- [ ] ❗ **Блок 4: Контент — часть 1** (~100-150 слайдов)
  - [ ] Минимум 3 инсайта с вау-эффектом
  - [ ] Каждый инсайт = открытие + пример + применение
  - [ ] Нативное упоминание составляющих продукта
  - [ ] Опенлупы к бонусам и продаже

- [ ] ❗ **Блок 5: Актуализация перед продажей** (~30-40 слайдов)
  - [ ] "Точка Б" — описание желаемого будущего
  - [ ] 2-3 эмоциональные истории трансформации
  - [ ] Опенлупы перед продажей

- [ ] ❗ **Блок 6: Продажа — Окно 1** (~50-70 слайдов)
  - [ ] Естественный переход от контента
  - [ ] 5 составляющих продукта с ценностью
  - [ ] Демонстрация полной стоимости
  - [ ] Гарантия результата
  - [ ] Минимум 2 ценовых сброса
  - [ ] Бонусы с ценностью
  - [ ] Срочность (время/места)
  - [ ] Пошаговая инструкция по оплате + QR

- [ ] **Блок 7: Контент — часть 2** (~50-80 слайдов)
  - [ ] Дополнительный ценный контент
  - [ ] Практические упражнения

- [ ] ❗ **Блок 8: Продажа — Окно 2** (~30-40 слайдов)
  - [ ] Рациональные аргументы (ROI, экономия)
  - [ ] Повторение предложения

- [ ] **Блок 9: Разбор возражений** (~30-50 слайдов)
  - [ ] Отработка минимум 5 возражений
  - [ ] Каждое возражение = признание + решение + кейс

- [ ] ❗ **Блок 10: Финал — Окно 3** (~10-20 слайдов)
  - [ ] Последний призыв к действию
  - [ ] Напоминание об ограничении
  - [ ] Риск упущенной возможности
  - [ ] Крупный QR-код на заявку

## 2. КАЖДЫЙ СЛАЙД

### 2.1 Правило "Одна пуля"

- [ ] ❗ Каждый слайд = ОДНА мысль (не 5-10 пунктов)
- [ ] Максимум 1-3 коротких тезиса на слайде
- [ ] Нет простыней текста
- [ ] Слайд читается за 3 секунды

### 2.2 Самодостаточность

- [ ] ❗ Слайд понятен БЕЗ голоса спикера
- [ ] Заголовок раскрывает смысл (не "Проблема", а "Почему 90% не зарабатывают")
- [ ] Буллеты — конкретные, не абстрактные

### 2.3 Продаёт следующий

- [ ] ❗ Каждый слайд создаёт желание узнать, что дальше
- [ ] Используются техники:
  - [ ] Незаконченная мысль: "Но есть одна проблема..."
  - [ ] Обещание: "Сейчас покажу как..."
  - [ ] Интрига: "И тут произошло кое-что интересное..."

---

## 3. КОНТЕНТ

### 3.1 Инсайты и ценность

- [ ] ❗ Каждый контентный блок даёт инсайт ("Вау, так можно было?")
- [ ] Контент сразу применим (можно использовать после вебинара)
- [ ] Контент меняет картину мира (не банальщина)
- [ ] Нет "воды" — только конкретика

### 3.2 Кейсы и доказательства

- [ ] ❗ Есть кейсы с конкретными цифрами
- [ ] Есть скриншоты/визуальные подтверждения
- [ ] Кейсы из разных ниш (релевантно разным сегментам ЦА)

### 3.3 Простота языка

- [ ] ❗ Текст понятен 10-летнему ребёнку (правило бабушки)
- [ ] Нет терминов без объяснения
- [ ] Сложное объясняется через метафоры:
  - [ ] Нейросеть → "умный помощник"
  - [ ] Промпт → "рецепт для робота"
  - [ ] Автоматизация → "конвейер Форда"

### 3.4 Нативная продажа

- [ ] Составляющие продукта упоминаются в контенте
- [ ] Возражения отрабатываются через истории (не в лоб)
- [ ] Продукт = естественное решение проблемы из контента

---

## 4. ЭМОЦИИ

### 4.1 Эмоциональные качели

- [ ] ❗ Есть смена эмоций: юмор → инсайт → гордость → вдохновение
- [ ] Нет монотонности (не только контент, не только продажа)

### 4.2 Юмор

- [ ] ❗ Есть юмор в вступлении
- [ ] Есть юмор в переходах между блоками
- [ ] Есть самоирония в историях
- [ ] Юмор уместный и не натянутый

### 4.3 Истории

- [ ] ❗ Минимум 2-3 истории трансформации
- [ ] Истории вызывают эмоции (не сухие факты)
- [ ] Истории связаны с продуктом
- [ ] Есть личная история спикера

### 4.4 Точка Б

- [ ] ❗ Зритель "помещён" в желаемое будущее
- [ ] Используются вопросы: "Представьте себе..."
- [ ] Описано и эмоциональное, и материальное состояние

---

## 5. ОПЕНЛУПЫ И УДЕРЖАНИЕ

### 5.1 Количество

- [ ] ❗ Минимум 5-7 опенлупов на весь вебинар
- [ ] Опенлуп в начале (интрига плана)
- [ ] Опенлупы перед каждым блоком
- [ ] Опенлуп перед продажей (эксклюзив в конце)

### 5.2 Качество

- [ ] Опенлупы конкретные (не "скоро узнаете кое-что")
- [ ] Используются формулы:
  - [ ] "Чуть позже я покажу, как [результат]"
  - [ ] "В конце я раскрою секрет, который [обещание]"
  - [ ] "Оставайтесь, в финале будет бомба"

### 5.3 ВАУ-эффект

- [ ] ❗ ВАУ-момент каждые 3-5 минут (5-7 слайдов)
- [ ] Типы ВАУ использованы:
  - [ ] Шокирующая статистика
  - [ ] Контринтуитивный факт
  - [ ] Демонстрация в реальном времени
  - [ ] Личная история
  - [ ] Разрыв шаблона

---

## 6. ПРОДАЖА

### 6.1 Три окна продаж

- [ ] ❗ Окно 1 — эмоциональное (после контента, на пике вдохновения)
- [ ] ❗ Окно 2 — рациональное (ROI, расчёты, логика)
- [ ] ❗ Окно 3 — финальное (срочность, последний шанс)

### 6.2 Раскладка ценности

- [ ] ❗ Продукт разложен на 5 составляющих
- [ ] Каждая составляющая = ценность + стоимость
- [ ] Общая воспринимаемая ценность = 5x от цены
- [ ] Показана "сумма всего" перед ценой

### 6.3 Ценовые сбросы

- [ ] ❗ Минимум 2 ценовых сброса
- [ ] Каждый сброс обоснован (почему скидка)
- [ ] Финальная цена выглядит выгодной на фоне ценности

### 6.4 Бонусы

- [ ] ❗ Минимум 3 бонуса с ценностью
- [ ] Бонусы анонсированы через опенлупы в контенте
- [ ] Есть главный бонус (удержание до конца)
- [ ] Общая ценность бонусов = 3-5x от цены продукта

### 6.5 Гарантия

- [ ] ❗ Гарантия чётко сформулирована
- [ ] Условия гарантии понятны
- [ ] Гарантия визуализирована

### 6.6 Срочность и дефицит

- [ ] ❗ Есть ограничение по времени
- [ ] Есть ограничение по местам
- [ ] Таймер/счётчик показан
- [ ] Срочность обоснована (не "просто так")

### 6.7 CTA (призыв к действию)

- [ ] ❗ CTA чёткий: что делать прямо сейчас
- [ ] QR-код крупный и читаемый
- [ ] Ссылка простая для запоминания
- [ ] Пошаговая инструкция по оплате
- [ ] Описано что произойдёт после оплаты

---

## 7. ВОЗРАЖЕНИЯ

### 7.1 Основные возражения отработаны

- [ ] ❗ "Дорого" — показан ROI, раскладка ценности
- [ ] ❗ "Нет времени" — показано сколько реально займёт
- [ ] ❗ "Не уверен что получится" — кейсы похожих людей
- [ ] ❗ "Подумаю" — срочность, риск упущенной возможности
- [ ] "Уже пробовал — не работает" — чем это отличается

### 7.2 Формат отработки

- [ ] Возражение признаётся ("Понимаю, это важный вопрос")
- [ ] Переформулирование ("На самом деле вопрос в том...")
- [ ] Решение (как продукт решает)
- [ ] Кейс/история как доказательство

---

## 8. ВИЗУАЛ И ОФОРМЛЕНИЕ

### 8.1 Каждый слайд

- [ ] Есть описание визуала в поле ВИЗУАЛ:
- [ ] Визуал поддерживает смысл (не просто декор)
- [ ] Цифры → графики/диаграммы
- [ ] Кейсы → скриншоты/фото
- [ ] Процессы → схемы

### 8.2 Социальные доказательства

- [ ] Скриншоты отзывов
- [ ] Фото учеников/клиентов
- [ ] Видео-отзывы (ссылки)
- [ ] Логотипы компаний/наград

### 8.3 Оформление текста

- [ ] Заголовки интригующие (не описательные)
- [ ] Буллеты короткие (1 строка)
- [ ] Эмодзи используются уместно (✅ ❌ 🔥)

---

## 9. ИНТЕРАКТИВ

### 9.1 Вовлечение аудитории

- [ ] ❗ Интерактив каждые 5-7 минут
- [ ] Типы интерактива:
  - [ ] "Напишите в чат [слово]"
  - [ ] "Поднимите руку, кто..."
  - [ ] "Кто из вас сталкивался с...?"
  - [ ] Опрос/голосование

### 9.2 Качество интерактива

- [ ] Интерактив связан с контентом (не для галочки)
- [ ] Понятно, зачем это делать (ценность для зрителя)
- [ ] Есть реакция на ответы ("Отлично, вижу...")

---

## 10. ТЕХНИЧЕСКИЙ ФОРМАТ

### 10.1 Формат файла

- [ ] Слайды в формате `===SLIDE===`
- [ ] Каждый слайд содержит:
  - [ ] ТИП: (титульный/контент/фото/чат/бонус/cta/промпт)
  - [ ] ЗАГОЛОВОК:
  - [ ] ТЕКСТ: (буллеты или пусто)
  - [ ] СПИКЕР: (речь — идёт НА слайд!)
  - [ ] ВИЗУАЛ: (для дизайнера — в заметки)

### 10.2 Правила полей

- [ ] СПИКЕР и ТЕКСТ — на слайде
- [ ] ВИЗУАЛ — только в заметках для дизайнера
- [ ] Разделители `===SLIDE===` на отдельных строках

## 🔧 Что делать, если не проходит проверку

### Слабые опенлупы
→ Добавь конкретики: "Через 15 минут покажу, как Маша заработала 500К за месяц"

### Провисание внимания
→ Вставь вопрос, опрос, историю или демонстрацию

### Слабая продажа
→ Усиль раскладку ценности, добавь социальные доказательства

### Мало эмоций
→ Вплети метафоры, юмор, личные истории

### Возражения не отработаны
→ Добавь истории клиентов с похожей ситуацией

### Слайды "не продают" следующий
→ Добавь в конец: "Но есть одна проблема...", "Сейчас покажу как..."
