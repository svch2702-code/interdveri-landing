# Методология работы с бонусами на вебинаре

Этот документ описывает, как создавать и использовать бонусы для продающих вебинаров.

---

## Зачем нужны бонусы

Бонусы выполняют 3 функции:

1. **Опен-лупы** — удерживают зрителя до конца эфира ("досмотрите — расскажу про бонус")
2. **Усиление оффера** — увеличивают воспринимаемую ценность предложения
3. **Эксклюзивность** — создают ощущение "только здесь и сейчас"

---

## Структура описания бонуса

Для каждого бонуса заполняй следующие поля:

```markdown
### Название бонуса
**Рыночная стоимость:** X ₽
**Что это:** Краткое описание (1-2 предложения)
**Результат для участника:** Что конкретно получит человек
**Для кого особенно ценен:** Какой сегмент ЦА оценит больше всего
**Когда анонсировать:** В какой момент вебинара упоминать
**Фраза для опен-лупа:** Готовая фраза для анонса
```

---

## Типы бонусов

### 1. Процессный бонус (спецпроект)
**Что это:** Дополнительная программа/формат работы (фокус-группа, практикум, сопровождение)

**Пример:** "Секретный спецпроект — закрытая фокус-группа по вайбкодингу"

**Особенности:**
- Высокая ценность (от 100 000 ₽)
- Ограничение по местам
- Личное участие спикера
- Главный бонус вебинара

### 2. Материальный бонус
**Что это:** Готовые материалы, шаблоны, чек-листы, инструменты

**Примеры:**
- "100 готовых промптов для бизнеса"
- "Шаблон воронки продаж"
- "Библиотека нейросотрудников"

**Особенности:**
- Средняя ценность (5 000 – 50 000 ₽)
- Можно использовать сразу
- Экономит время

### 3. Доступ/сервис
**Что это:** Доступ к платформе, сервису, подписка

**Примеры:**
- "Подписка на сервис создания продуктов"
- "Доступ к закрытому чату"
- "1 месяц в НейроКлубе"

**Особенности:**
- Ценность = стоимость подписки
- Триал для привлечения в экосистему

### 4. Экспертный бонус
**Что это:** Личная консультация, разбор, сессия со спикером

**Примеры:**
- "Персональный разбор вашего проекта"
- "Стратегическая сессия 1-на-1"

**Особенности:**
- Высокая ценность (от 30 000 ₽)
- Ограничение по количеству
- Максимальная эксклюзивность

## Как использовать бонусы во время эфира

### Момент 1: Начало вебинара (первые 5 минут)
**Цель:** Создать интригу, удержать до конца

**Что говорить:**
> "Кстати, в конце я расскажу про кое-что особенное. Это то, чего у нас никогда не было. Досмотрите до конца — не пожалеете."

**Не называть бонус конкретно!** Только интрига.

### Момент 2: После первого контентного блока (15-20 минут)
**Цель:** Напомнить, усилить интерес

**Что говорить:**
> "Помните, я говорил про что-то особенное? Скоро расскажу. Это связано с тем, что мы сейчас разбирали."

### Момент 3: Перед продающим блоком
**Цель:** Подготовить к восприятию ценности

**Что говорить:**
> "Сейчас я расскажу про специальное предложение. И там будет кое-что, чего вы точно не ожидаете. Но сначала — давайте разберём, что вы получите..."

### Момент 4: Во время продажи (после цены)
**Цель:** Усилить оффер, добавить ценность

**Порядок представления бонусов:**
1. Назвать основной продукт и цену
2. Добавить первый бонус: "Но это не всё. Вы также получаете..."
3. Добавить второй бонус: "И ещё..."
4. Добавить главный бонус: "А теперь самое важное..."
5. Показать общую ценность: "Итого ценность того, что вы получаете — X рублей"
6. Показать цену: "А вы платите только..."

### Момент 5: Закрытие (последние 10 минут)
**Цель:** Создать срочность

**Что говорить:**
> "Напоминаю: [главный бонус] доступен только тем, кто оставит заявку сегодня. Завтра этого уже не будет."

## Расчёт воспринимаемой ценности

**Правило:** Общая ценность бонусов должна быть в 3-5 раз выше цены продукта.

**Пример для продукта за 50 000 ₽:**

| Бонус | Ценность |
|-------|----------|
| Разбор проекта | 30 000 ₽ |
| Шаблоны и материалы | 15 000 ₽ |
| Доступ к сервису | 10 000 ₽ |
| Спецпроект (главный) | 150 000 ₽ |
| **Итого** | **205 000 ₽** |

Воспринимаемая ценность: 205 000 ₽
Цена: 50 000 ₽
Множитель: 4.1x ✓

## Как создать файл бонусов для вебинара

1. Создай файл `Бонусы_[НАЗВАНИЕ_ВЕБИНАРА].md` в папке `context/`
2. Опиши каждый бонус по структуре выше
3. Рассчитай общую ценность
4. Пропиши моменты анонсов и готовые фразы

**Пример:** `Бонусы_АНТИТРЕНДЫ.md` — бонусы для практикума "АНТИТРЕНДЫ 2026"

## Фразы для опен-лупов (шаблоны)

### Интрига в начале:
- "В конце покажу кое-что, чего у нас никогда не было"
- "Досмотрите до конца — там будет сюрприз"
- "Я готовил это несколько недель специально для вас"

### Напоминание в середине:
- "Помните про сюрприз? Скоро..."
- "Кстати, то, о чём я говорил — это связано с [текущей темой]"

### Анонс бонуса:
- "А теперь — то, ради чего многие досмотрели до конца"
- "Сейчас я расскажу про [бонус], который стоит X рублей, но вы получите его бесплатно"

### Срочность:
- "Это только для тех, кто на эфире. Завтра этого не будет"
- "Мест всего X. Первыми попадают те, кто оставит заявку сейчас"

## Чек-лист перед вебинаром

- [ ] Файл бонусов создан (`Бонусы_[НАЗВАНИЕ].md`)
- [ ] Каждый бонус описан по структуре
- [ ] Рассчитана общая ценность (должна быть 3-5x от цены)
- [ ] Прописаны моменты анонсов
- [ ] Готовы фразы для опен-лупов
- [ ] Есть главный бонус (самый ценный, для удержания до конца)
- [ ] Есть ограничение (по времени или местам)
