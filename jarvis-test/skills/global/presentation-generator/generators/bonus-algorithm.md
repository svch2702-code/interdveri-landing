## Формула промптов: CRAFT

Все промпты в бонусах создаются по формуле **CRAFT** и оформляются как **шаблоны** — пользователь подставляет свои данные в `[плейсхолдеры]`.

| Буква | Что значит | Пример |
|-------|-----------|--------|
| **C** — Context | Контекст ситуации | "Я хочу создать [тип продукта] для [ниша]" |
| **R** — Role | Роль для нейросети | "Ты — опытный [роль]" |
| **A** — Action | Конкретное действие | "Проанализируй / Составь / Создай..." |
| **F** — Format | Формат результата | "Результат оформи в виде [таблица/список/план]" |
| **T** — Tone | Тон и ограничения | "Простым языком, без жаргона, для новичков" |

**Каждый промпт:**
- Решает одну конкретную задачу
- Содержит `[плейсхолдеры в квадратных скобках]` для подстановки данных
- Оформлен в `prompt-block` с кнопкой "Скопировать"
- Даёт результат за 10-20 минут
- Понятен полному новичку

## Формат инструкций: 1-2-3

Каждый шаг/раздел в бонусе описывается по единой технологии:

```
ШАГ X: [Название шага]

1. ЧТО ДЕЛАЕМ — описание действия простым языком
2. ПРОМПТ — готовый шаблонный промпт по формуле CRAFT (в prompt-block с кнопкой копирования)
3. ЗАЧЕМ — почему этот шаг важен, что будет если пропустить
4. РЕЗУЛЬТАТ — что человек получает после выполнения (конкретный артефакт)
```

**Пример:**
```
ШАГ 1: Формулируем идею

1. ЧТО ДЕЛАЕМ: Открываем ChatGPT и описываем что хотим создать
2. ПРОМПТ: [шаблонный промпт по CRAFT с плейсхолдерами]
3. ЗАЧЕМ: Без чёткой идеи нейросеть сделает что-то размытое
4. РЕЗУЛЬТАТ: Документ с описанием идеи на 1 страницу
```

## Структура бонуса: обязательные блоки

### Начало
1. Главный заголовок (h1.bonus-main-title) — ёмкий, 3-5 слов
2. Подзаголовок (p.subtitle) — развёрнутая выгода
3. Оглавление (bonus-toc) — каждый пункт как оффер
4. Вступление (150-300 слов) — личная история от лица Дмитрия

### Середина
5. 5-7 содержательных секций с h2
   - Каждая секция в формате 1-2-3 (что делаем → промпт → зачем → результат)
   - Промпты по формуле CRAFT с кнопкой копирования
   - Ссылки на инструменты с иконками при каждом упоминании

### Конец (ОБЯЗАТЕЛЬНО)
6. **Резюме** — краткая выжимка: что человек узнал, какие шаги прошёл, что теперь умеет
7. **Нативная интрига** — посыл в духе:
   > "Один — это круто. Ты уже можешь сделать [результат]. Но в компании людей, которые в этом разбираются, всегда эффективнее. Когда рядом те, кто уже прошёл этот путь — ты двигаешься быстрее, не наступаешь на грабли, и получаешь обратную связь на каждом шаге."
   - Это **нативно** подводит к мысли, что стоит учиться у нас
   - БЕЗ прямой продажи — просто мысль "в компании круче"
8. CTA-блок — кнопки на бота и канал
9. Meta-дисклеймер + подпись автора + соцсети

## 5 шагов создания бонуса

### ШАГ 1: Исследование темы
**Что делаю:** Ищу реальные данные в интернете
- Цифры и статистики по теме (в рублях!)
- Реальные кейсы и примеры
- Актуальные инструменты и сервисы (с реферальными ссылками)
- Рыночные цены (если тема связана с заработком)

**Правило:** Никаких выдуманных цифр. Только то, что можно проверить. USD → RUB. Все для аудитории из России.

### ШАГ 2: Составление структуры
**Что делаю:** Определяю 5-7 секций + подбираю компоненты

Каждая секция планируется в формате 1-2-3:
- Что делаем на этом шаге
- Какой промпт нужен (по формуле CRAFT)
- Зачем этот шаг
- Что человек получает в результате

Компоненты по ситуации:
- `bonus-highlight` — ключевые инсайты (💡 или ⚠️)
- `data-table` — для цифр, сравнений, цен
- `bonus-comparison` — блоки "Было → Стало"
- `prompt-block` — **ОБЯЗАТЕЛЬНО на каждом шагу** — промпты CRAFT с кнопкой копирования
- `direction-block` — для направлений/вариантов
- `scenario-block` — для финансовых расчётов
- `step-block` — для пошаговых планов
- `case-block` — для кейсов
- `pros-cons` — плюсы/минусы
- `stat-pill` — метрики в плашках
- `tool-link` — ссылки на инструменты с иконками

### ШАГ 3: Написание контента
**Стиль:**
- От первого лица Дмитрия Ледовских
- Простой язык — **настолько простой, чтобы разобрался даже ребёнок**
- Разговорный тон: "Короче", "Ребят", "А что если..."
- Короткие абзацы (1-3 предложения)
- Конкретные цифры (минимум 5-7 на весь материал, все в рублях)
- Без канцелярита, без жаргона, без пассивного залога
- При упоминании инструментов — всегда ссылка + иконка
- Промпты на каждом шагу по формуле CRAFT — шаблонные, с плейсхолдерами
- В конце: резюме + нативная интрига про "в компании эффективнее"

**Запрещено:**
- Служебные заголовки ("Эмоциональное вступление", "Мягкий мост к продукту")
- Markdown-разметка (только HTML)
- Выдуманные цифры и преувеличения
- Общие фразы без конкретики
- Мета-комментарии ("Вот бонус:", "Готово!")
- Суммы в долларах (только рубли)

### ШАГ 4: Сборка HTML
**Что делаю:** Оборачиваю контент в тёмный шаблон

Шаблон включает:
- Шрифт Inter (Google Fonts)
- Фон #0a0a0f, текст #e0e0e8
- Градиенты purple→blue→green (#a78bfa → #818cf8 → #60a5fa → #34d399)
- Анимация gradientShift для заголовка
- Fade-in анимация при скролле (IntersectionObserver)
- Smooth scroll для оглавления
- Полная адаптивность (breakpoints: 560px, 480px)
- Max-width: 720px

Все стили — inline в `<style>`. Все скрипты — inline в `<script>`. Никаких внешних зависимостей кроме Google Fonts.

### ШАГ 5: Сохранение файла
**Формат имени:** `XX. Название темы.html`
- XX = следующий номер по порядку в папке Бонусы
