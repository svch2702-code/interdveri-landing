# Формат текстового файла для Google Slides

Этот документ описывает формат текстового файла, который автоматически конвертируется в Google Slides презентацию через скрипт `convert_google.sh`.

---

## 🎯 Общие правила

1. Слайды разделяются маркером `===SLIDE===`
2. Каждый слайд начинается с указания типа: `ТИП: название_типа`
3. Все поля пишутся с ЗАГЛАВНЫХ БУКВ и заканчиваются двоеточием
4. После двоеточия идёт содержимое (может быть многострочным)
5. Пустые строки между полями допустимы

---

## 📋 Типы слайдов

| № | Тип | Когда использовать | Плейсхолдеры |
|---|-----|-------------------|--------------|
| 1 | титульный | Первый слайд урока/вебинара | НОМЕР_УРОКА + ЗАГОЛОВОК |
| 2 | контент | Основная информация (буллеты) | ЗАГОЛОВОК + ТЕКСТ |
| 3 | фото | Демонстрация с картинкой | ЗАГОЛОВОК + ТЕКСТ + ВИЗУАЛ |
| 4 | чат | Интерактив, вопросы аудитории | ЗАГОЛОВОК + ТЕКСТ или ПРИЗЫВ |
| 5 | промпт | Готовый промпт для копирования | ЗАГОЛОВОК + ПРОМПТ |
| 6 | бонус | Анонс бонуса | ЗАГОЛОВОК + ТЕКСТ |
| 7 | cta | QR-код, призыв к действию | ЗАГОЛОВОК + ТЕКСТ |
| 8 | заключение | Итоговая мысль | ЗАГОЛОВОК |

---

## 📝 Поля для каждого слайда

### Общие поля (доступны для всех типов):

| Поле | Назначение | Куда попадает |
|------|------------|---------------|
| `ТИП:` | Тип слайда (обязательно!) | Выбор шаблона |
| `ЗАГОЛОВОК:` | Заголовок слайда | На слайд |
| `ТЕКСТ:` | Буллеты или основной текст | На слайд |
| `СПИКЕР:` | Речь спикера (развёрнутая) | В заметки |
| `ВИЗУАЛ:` | Описание для дизайнера | В заметки |

### Специальные поля (для конкретных типов):

| Поле | Для каких типов | Назначение |
|------|-----------------|------------|
| `НОМЕР_УРОКА:` | титульный | "Урок 1", "День 2" (в фиолетовом блоке) |
| `ПРИЗЫВ:` | чат | Короткий call-to-action ("ХОЧУ!") |
| `ПРОМПТ:` | промпт | Готовый промпт для копирования |

## 🎨 Примеры слайдов

### 1. Титульный слайд (разделитель темы/блока)

**Когда использовать:**
- Первый слайд урока, вебинара, воркшопа
- **Начало нового блока внутри презентации** (не только в начале!)
- Переход к новой теме
- Разделитель между частями вебинара

**Примеры НОМЕР_УРОКА:**
- `Урок 1` — для уроков
- `День 2` — для вебинаров
- `Блок 3: Практика` — для блоков внутри презентации
- `Часть 2` — для разделения частей
- `Тема: Продвинутый уровень` — для новых тем

**Структура:**
```
===SLIDE===
ТИП: титульный
НОМЕР_УРОКА: День 2
ЗАГОЛОВОК: Создание первого AI-агента
```

**Что получится:**
- Фиолетовый блок сверху: "День 2" (или любой другой текст)
- Крупный заголовок внизу: "Создание первого AI-агента"

**Важно:** Этот тип слайда можно использовать **НЕ ТОЛЬКО в начале**, но и внутри презентации для разделения тем/блоков.

**Плейсхолдеры в шаблоне:**
- `{{ LESSON_NUMBER }}` → НОМЕР_УРОКА (любой текст для разделителя)
- `{{ TITLE }}` → ЗАГОЛОВОК

---

### 2. Контентный слайд (основной информационный)

**Когда использовать:** Для передачи информации через буллеты, списки, пункты.

**Структура:**
```
===SLIDE===
ТИП: контент
ЗАГОЛОВОК: Как будет проходить обучение
ТЕКСТ:
• Онлайн-формат
• 2 месяца
• Практические задания
• Обратная связь и живые разборы
• Постоянные шаблоны и шпаргалки
СПИКЕР: Сейчас расскажу подробнее про формат обучения. У нас будет полностью онлайн-формат...
```

**Что получится:**
- Заголовок вверху слайда (салатовый)
- Буллеты на слайде
- СПИКЕР идёт в заметки (НЕ на слайд!)

**Правила для ТЕКСТ:**
- Буллеты: 1-7 пунктов (при Progressive Reveal начинай с одного)
- Каждый пункт — 1 строка (максимум 2)
- Используй • или - для буллетов
- Слайд может быть БЕЗ поля ТЕКСТ (только ЗАГОЛОВОК) — для акцентов и метафор
- **ВАЖНО: Простой язык БЕЗ жаргона!**
  - Разговорный, но деловой стиль
  - Понятны БЕЗ спикера (самодостаточные)
  - Понятно 10-летнему ребёнку
  - БЕЗ жаргона постов ("короче", "классный", "крутой")

**Примеры правильных буллетов:**
- ✅ "AI читает промпт и генерирует ответ за секунды"
- ✅ "Google AI Studio — бесплатный инструмент для работы"
- ✅ "Вы создаёте продукт БЕЗ кода — только промпты"

**Примеры неправильных буллетов:**
- ❌ "Искусственный интеллект обрабатывает запрос" (канцелярит)
- ❌ "Использование инструмента" (непонятно что делать)
- ❌ "Короче, делаем так..." (жаргон — только для постов!)
- ❌ "AI" (слишком коротко, нет контекста)

**Плейсхолдеры в шаблоне:**
- `{{ TITLE }}` → ЗАГОЛОВОК
- `{{ BULLETS }}` → ТЕКСТ

---

### 3. Слайд с фото (текст + картинка)

**Когда использовать:** Демонстрация, примеры, кейсы с визуализацией.

**Структура:**
```
===SLIDE===
ТИП: фото
ЗАГОЛОВОК: Мы собрали все, чего не хватало другим курсам
ТЕКСТ:
живое наставничество, реальные инструменты, практику и окружение.

Потому что мы не учим просто «про нейросети» — мы учим зарабатывать и жить по-новому.
ВИЗУАЛ: Картинка робота с планшетом
```

**Что получится:**
- Заголовок вверху
- Текст слева
- Картинка справа (вставляется вручную после конвертации)
- ВИЗУАЛ в заметках для дизайнера

**Правила для ТЕКСТ:**
- Может быть как буллетами, так и сплошным текстом
- 2-4 абзаца максимум

**Плейсхолдеры в шаблоне:**
- `{{ TITLE }}` → ЗАГОЛОВОК
- `{{ BULLETS }}` → ТЕКСТ

**Важно:** Картинку вставляешь вручную в Google Slides после конвертации.

**Вариант без текста (только заголовок + картинка):**
```
===SLIDE===
ТИП: фото
ЗАГОЛОВОК: Что будем отправлять клиенту?
ВИЗУАЛ: Мем с мужчиной в задумчивости
```

---

### 4. Интерактивный слайд (чат)

**Когда использовать:** Каждые 5-7 минут для вовлечения аудитории.

**Вариант 1: Развёрнутые вопросы**
```
===SLIDE===
ТИП: чат
ЗАГОЛОВОК: Напишите в чат:
ТЕКСТ:
Что может помешать вам применить полученные знания?

Какие у вас есть страхи и сомнения?
СПИКЕР: Давайте сейчас обсудим в чате, какие барьеры вы видите...
```

**Вариант 2: Короткий призыв**
```
===SLIDE===
ТИП: чат
ПРИЗЫВ: ХОЧУ!
СПИКЕР: Напишите "ХОЧУ" в чат, если хотите получить бонус в конце эфира
```

**Что получится:**
- Заголовок (если есть)
- Вопросы ИЛИ короткий призыв крупно
- Боковая панель "ЧАТ" (уже в шаблоне)

**Правила для ПРИЗЫВ:**
- 1-5 слов
- КАПСОМ или яркое выделение
- Примеры: "ХОЧУ!", "Напишите ДА", "Готовы?"

**Плейсхолдеры в шаблоне:**
- `{{ TITLE }}` → ЗАГОЛОВОК (опционально)
- `{{ BULLETS }}` → ТЕКСТ (для вопросов)
- `{{ CTA }}` → ПРИЗЫВ (для короткого призыва)

---

### 5. Слайд с промптом

**Когда использовать:** Показываем готовый промпт для копирования.

**Структура:**
```
===SLIDE===
ТИП: промпт
ЗАГОЛОВОК: Промпт 1: Упаковка в продукт
ПРОМПТ:
Я работаю с заранее выбранной ИИ-услугой: [вставь свою услугу — например: создание ИИ-продавца / ИИ-сотрудника].
На основе этой услуги разработай 3 варианта ее коммерческой упаковки для бизнеса:

БАЗОВАЯ ИНФОРМАЦИЯ: Название услуги (как ее назвать для клиентов), Главную боль бизнеса, которую решает, Конкретный результат для клиента (в цифрах/времени)

КОММЕРЧЕСКИЕ ПАРАМЕТРЫ: Стоимость услуги (ориентируйся на диапазон цен на рынке), Время выполнения проекта, Окупаемость для клиента (во что обойдется, сколько сэкономит)
```

**Что получится:**
- Заголовок сверху (опционально)
- Промпт в жёлтой/зелёной рамке
- Моноширинный шрифт (готов к копированию)

**Правила для ПРОМПТ:**
- Формула RAFT (Role, Audience, Format, Task)
- Переменные в {фигурных скобках} или [квадратных]
- Структурированный, с абзацами
- Готов к копированию (copy-paste ready)

**Плейсхолдеры в шаблоне:**
- `{{ TITLE }}` → ЗАГОЛОВОК (опционально)
- `{{ PROMPT }}` → ПРОМПТ

---

### 6. Слайд с бонусом

**Когда использовать:** Анонс бонуса, удержание до конца, опенлуп.

**Структура:**
```
===SLIDE===
ТИП: бонус
ЗАГОЛОВОК: Бонус: 50 готовых промптов
ТЕКСТ:
• Промпты для 10 ниш
• Экономия 10 часов
• Стоимость: 5 000 ₽
```

**Что получится:**
- Заголовок (название бонуса)
- 3-4 пункта о ценности

**Правила для бонуса:**
- Название бонуса в ЗАГОЛОВОК
- ТЕКСТ: что внутри (2-4 пункта)
- Обязательно указать ценность (что получит, сколько сэкономит)

**Плейсхолдеры в шаблоне:**
- `{{ TITLE }}` → ЗАГОЛОВОК
- `{{ BULLETS }}` → ТЕКСТ

---

### 7. Призыв к действию с QR (CTA)

**Когда использовать:** Отправка на ссылку, регистрация, бронирование.

**Структура:**
```
===SLIDE===
ТИП: cta
ЗАГОЛОВОК: Специальное предложение!
ТЕКСТ: Забронируй место и получи лучшие условия
```

**Что получится:**
- Заголовок
- Подзаголовок/описание
- QR-код справа (вставляется вручную)

**Плейсхолдеры в шаблоне:**
- `{{ TITLE }}` → ЗАГОЛОВОК
- `{{ SUBTITLE }}` → ТЕКСТ

**Важно:** QR-код вставляешь вручную в Google Slides.

---

### 8. Заключительная мысль

**Когда использовать:** Итоговый слайд, главная мысль, вывод.

**Структура:**
```
===SLIDE===
ТИП: заключение
ЗАГОЛОВОК: Как понять, что в продукте обязательно, а что излишне
```

**Что получится:**
- Заголовок крупно по центру
- Красивый градиентный фон

**Правила:**
- Одно предложение
- Сильная мысль
- Может содержать выделенные слова (выделяешь вручную после конвертации)

**Плейсхолдеры в шаблоне:**
- `{{ CONCLUSION }}` → ЗАГОЛОВОК

## 🎭 Паттерны слайдов (техники подачи)

Помимо типов слайдов, есть **техники подачи** — как оформлять последовательности слайдов.
Подробнее: [МЕТОДОЛОГИЯ_ПРЕЗЕНТАЦИЙ.md](МЕТОДОЛОГИЯ_ПРЕЗЕНТАЦИЙ.md)

### Progressive Reveal (постепенное раскрытие)

Список из 3+ пунктов разбивается на серию слайдов. Каждый слайд добавляет один пункт:

```
===SLIDE===
ТИП: контент
ЗАГОЛОВОК: Как подключить проект к GitHub
ТЕКСТ:
Говорим Claude: «Подключи проект к GitHub»

===SLIDE===
ТИП: контент
ЗАГОЛОВОК: Как подключить проект к GitHub
ТЕКСТ:
Говорим Claude: «Подключи проект к GitHub»
Claude выполнит все команды сам

===SLIDE===
ТИП: контент
ЗАГОЛОВОК: Как подключить проект к GitHub
ТЕКСТ:
Говорим Claude: «Подключи проект к GitHub»
Claude выполнит все команды сам
Заходим на github.com — видим свой проект
СПИКЕР: Одна фраза — и проект в облаке.
```

### Слайд-акцент (только заголовок)

```
===SLIDE===
ТИП: контент
ЗАГОЛОВОК: Вам НЕ нужно учить HTML.
Claude знает его идеально
```

```
===SLIDE===

```

```
===SLIDE===
```

```
ЗАГОЛОВОК: Вот что получается в результате
```

## ⚠️ Частые ошибки

| Ошибка | Правильно |
|--------|-----------|
| `Тип: контент` | `ТИП: контент` (ЗАГЛАВНЫМИ) |
| `заголовок: текст` | `ЗАГОЛОВОК: текст` |
| Нет `===SLIDE===` | Обязательно разделяй слайды маркером |
| СПИКЕР на слайде | СПИКЕР идёт ТОЛЬКО в заметки! |
| Длинные буллеты (3-4 строки) | Буллеты короткие (1-2 строки) |
| Промпт не готов к копированию | Промпт должен быть copy-paste ready |

## 🚀 Конвертация: текст → Google Slides

После создания текстового файла:

```bash
cd "/Users/macbookledovskih/Documents/Projects/Помощник/Разработка презентаций и уроков"

# Дизайн по умолчанию:
./convert_google.sh "файл.txt" "Название презентации"

# С выбором дизайна:
./convert_google.sh "файл.txt" "Название презентации" -d vibecoding
```

**Доступные дизайны** (config.json → templates):
| Дизайн | Описание |
|--------|----------|
| `default` | Основной шаблон вебинаров |
| `vibecoding` | Шаблон для курса по вайбкодингу |

**Результат:**
- Автоматически создаётся презентация в Google Slides
- Все плейсхолдеры заменяются на реальный текст
- СПИКЕР и ВИЗУАЛ попадают в заметки
- Получаешь ссылку на готовую презентацию

**После конвертации (вручную):**
1. Вставить картинки (для слайдов типа "фото")
2. Вставить QR-коды (для слайдов типа "cta")
3. Выделить слова цветом (если нужно)
4. Проверить форматирование

## 📥 Обратный конвертер: Google Slides → текст

Извлекает текст из существующей Google Slides презентации в формат `===SLIDE===`:

```bash
cd "/Users/macbookledovskih/Documents/Projects/Помощник/Разработка презентаций и уроков"

# Вся презентация:
./extract_google.sh "https://docs.google.com/presentation/d/ID/edit"

# Конкретные слайды:
./extract_google.sh ID --slides 135-215

# Точечный выбор:
./extract_google.sh ID --slides 1,3,7-10,50-80

# Разбить большую презентацию на файлы по 30 слайдов:
./extract_google.sh ID --chunk 30

# Комбинация: диапазон + разбивка:
./extract_google.sh ID --slides 100-300 --chunk 30 -o вебинар.txt
```

**Результат:**
- Текстовый файл в формате `===SLIDE===`
- Заголовки и текст извлекаются из слайдов
- СПИКЕР и ВИЗУАЛ извлекаются из заметок
- Файл можно отредактировать и конвертировать обратно

**Полный цикл переноса в новый дизайн:**
```
1. Извлечь:       ./extract_google.sh ID_СТАРОЙ -o slides.txt
2. Отредактировать slides.txt (при необходимости)
```
