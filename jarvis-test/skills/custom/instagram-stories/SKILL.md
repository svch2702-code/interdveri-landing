# Скилл: Instagram Stories с текстом на фото

## Когда активировать
Клиент присылает фото и просит: "сделай сторис", "наложи текст", "подготовь stories", "текст на фото для инстаграм"

## Что делает скилл
Принимает фото от клиента, придумывает короткий текст в его стиле, накладывает на фото с правильной типографикой, отдаёт готовые изображения 1080x1920

## Инструмент
Скрипт: ~/workspace/tools/make-stories.js
Зависимость: sharp (установить в /tmp/story-gen/node_modules/)
Шрифты: ~/projects/carousel-bot/fonts/ (копируются в /tmp/fonts/ при запуске)

### Подготовка перед первым использованием в сессии
```bash
mkdir -p /tmp/fonts && cp ~/projects/carousel-bot/fonts/*.ttf /tmp/fonts/ 2>/dev/null
NPM_CONFIG_CACHE=/tmp/npm-cache npm install --prefix /tmp/story-gen sharp 2>/dev/null
mkdir -p /tmp/fc-conf && cat > /tmp/fc-conf/fonts.conf << 'EOF2'
<?xml version="1.0"?>
<!DOCTYPE fontconfig SYSTEM "fonts.dtd">
<fontconfig><dir>/tmp/fonts</dir><dir>/usr/share/fonts</dir></fontconfig>
EOF2
```

### Запуск
```bash
FONTCONFIG_PATH=/tmp/fc-conf node ~/workspace/tools/make-stories.js
```

## Instagram Safe Zones (1080x1920)
Подробно: knowledge/stories-safezone.md

- Верх (0-250px): ЗАПРЕТ — юзернейм, время, кнопка закрытия
- Низ (1640-1920px): ЗАПРЕТ — поле ответа, реакции
- Безопасная зона текста: y=250 до y=1640
- Боковые отступы: 60-72px

## Правила наложения текста

### Фото
- НИКОГДА не обрезать оригинал — вписывать целиком (contain, не cover)
- Пустое пространство заполнять доминантным цветом из фото (затемнённым)
- Формат: 1080x1920 (9:16), JPEG quality 95

### Текст
- Шрифт: Unbounded (Regular для подстрочника, Bold для основного, Black для акцента)
- Расположение: по умолчанию внизу, слева (leftPad=72)
- Нижняя граница текста: не ниже y=1610 (bottomPad=310)
- Без фоновых плашек/прямоугольников
- Градиент снизу: плавный переход от прозрачного к чёрному (opacity 0.85-0.88)
- Градиент занимает ~45% высоты изображения

### Цвет текста
- Подбирать под тон фото: тёплые фото — #FFF8F0, холодные — #FFFFFF

### Стиль текстов — Tone of Voice Дмитрия
ОБЯЗАТЕЛЬНО перед написанием текстов прочитай:
- knowledge/writing-style.md — бренд-код, тон голоса, паттерны речи
- knowledge/post-structure.md — структура, два регистра

Ключевые правила:
- Разговорный стиль: "Короче", "прям", "Вот в чём фишка"
- Ирония и самоирония, без пафоса
- От первого лица: "Я раньше...", "Я в этом не разбираюсь"
- Короткие фразы, 2-4 слова на строку
- Серия = история с нарастанием (завязка — кульминация — панчлайн)
- Контраст и двойной смысл в финале
- НЕ рекламный, НЕ мотивационный, НЕ менторский
- Как будто написал в Instagram между делом — живой и настоящий

### Иерархия текста
- line1 (подстрочник): font-size 38px, weight 400, opacity 0.75
- line2 (основной): font-size 54px, weight 700
- line3 (акцент/финал): font-size 54px, weight 900

## Шрифты (доступные)
- Unbounded (Regular/Bold/Black) — основной для сторис
- Montserrat — альтернатива для чистого стиля
- Caveat — рукописный, для личных историй
- BebasNeue — узкий display, для коротких акцентов
- Inter — для мелкого текста

## Как модифицировать make-stories.js
Каждый раз адаптируй массив stories[] под запрос:
- photo: путь к фото клиента
- line1, line2, line3: текст (line1 может быть пустым)
- textColor: цвет текста
- gradientStops: настройка градиента

## Пример процесса
1. Клиент шлёт фото + описывает ситуацию
2. Придумай текст (короткий, ироничный, в стиле Дмитрия)
3. Разбей на серию 2-4 сторис с нарастанием
4. Отредактируй stories[] в make-stories.js
5. Запусти скрипт
6. Отправь результат
7. Спроси нужны ли правки

## ЗАПРЕЩЕНО

- Размещать текст в запретных зонах Instagram
- Хайп-эмодзи в текстах на фото
